# Security Policy

## Supported Versions

Our platform is currently actively maintained. Security updates are applied to the latest branch as they become available.

| Version | Supported          |
| ------- | ------------------ |
| >= 1.0  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of AI Security Brief seriously. If you discover a security vulnerability in this project, please let us know immediately.

1. **Do not** open a public issue on GitHub for security vulnerabilities.
2. Email your findings to security@aisecuritybrief.com.
3. Please provide a detailed description of the vulnerability, including:
   - Type of issue (e.g., XSS, SQLi, CSRF, etc.)
   - Full URL(s) or file paths where the vulnerability exists
   - Proof of Concept (PoC) or exact steps to reproduce
   - Potential impact
4. We will acknowledge receipt of your vulnerability report within 48 hours and strive to resolve the issue promptly.

## Responsible Disclosure

We ask that you:
- Give us a reasonable time to investigate and mitigate an issue before making any information regarding the vulnerability public.
- Do not exploit the vulnerability beyond what is necessary to prove its existence.
- Do not utilize the vulnerability to access, modify, or destroy any user data.

## Bug Bounty

Currently, we do not have an official bug bounty program, but we greatly appreciate the efforts of security researchers who help keep our project safe. We will acknowledge your contribution (with your permission) once the issue is resolved.
